## Core Transformation

Вводим cheap rules-based классификатор траектории (`conversational` / `tool_heavy` / `mixed`) — без LLM, O(n) по истории. Добавляем `dispatch()` seam, который возвращает intent (`{runObserver, runOffloader}`), но не выполняет — orchestration живёт в A5. Hysteresis stateless: caller хранит `HysteresisState` (cache-friendly: меняется только при фактическом флипе). Phase разблокирует A5 и A6 (через seam) + даёт A2/A3 явный `current_class` для `CompactionContext`.

## Deviations

- **dispatch throws when `TRAJECTORY_CLASSIFIER=false && configuredClass === undefined`** — план не оговаривал. Добавлено как fail-fast guard: disabled-mode требует явно указанного класса. Альтернатива — фолбэк на `'mixed'` — была бы тише, но скрывала бы конфигурационную ошибку. Закреплено тестом.
- **`computeFeatures` дополнительно заполняет `avg_tool_result_size` и `cumulative_tokens`** — план перечислял только базовые 5 фич, но `ClassifierFeatures` (§2.4) содержит 7. Все 7 заполняем сразу, иначе тип неполный.

## Risks / Follow-ups

- **A2 заберёт `defaultByteCounter`** в shared `tokenCounter.ts` (`byteLengthOfContent`); inline-вариант в `classifierFeatures.ts` мигрирует. Не блокер.
- **A5 orchestrator** будет вызывать `compactWithOffload` / `maybeExtractObservations` на основе `dispatch()` результата; сейчас seam возвращает intent only.
