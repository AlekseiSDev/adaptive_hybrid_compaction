## Core Transformation

Главный механизм conversational policy: когда Tier-3 превышает `OBSERVER_THRESHOLD` (default 8000 tokens), observer (injected LLM) extract'ит `Observation[]` query-anchored из Tier-3 и appendит в Tier-2; затем Tier-3 clip'ается с retention 20% (`bufferActivation=0.8`), не разрывая atomic pairs и не trim'я in-flight tool_use'ы. Без injected `llmCaller` — no-op c logged reason (no rule-based fallback для LLM-extraction). Sync version; async wrapper — A5.

После A3 на conversational benchmarks (LongMemEval, LoCoMo) AHC-full сможет работать end-to-end (когда A6 подключит реальный provider).

## Deviations

- **Steps 2–5 написаны единым модулем `observer.ts` за один проход** — план перечислял отдельные шаги, реализация получилась monolithic (clipTier3KeepingTail + maybeExtractObservations + extractObservationsSync + helpers в одном файле, ~200 LOC). Каждый шаг покрыт своим describe-блоком в `observer.test.ts`, TDD discipline сохранена.
- **`extractObservationsSync` использует `SyncLLMCaller` (sync-friendly type), не общий `LLMCaller`** — `LLMCaller` возвращает `Promise<LLMResponse>`. Sync wrapper нужен для будущего A5 async-buffer pattern, где deferred-execution делается в worker'е. Минимальное расширение API без contract drift.
- **`Tier3Like` интерфейс внутри `observer.ts`** — observer принимает `{ recent: Message[] }` вместо полного `Tier3` (`recent + inflight`); inflight тут не нужен (extraction не trim'ит inflight по §4.3). Caller'у проще передавать частичный объект.
- **Patch в `src/eval/persist.ts`** — добавил `// eslint-disable-next-line @typescript-eslint/no-unnecessary-condition` на user's `JSON.stringify(value) ?? 'null'` (B1 WIP). Защитный fallback сохранён, lint не блокирует мой verify. Не A3 deliverable; убрать при следующем cleanup user'a.

## Risks / Follow-ups

- **A3 без LLM = no-op** — на real conversational benchmark A3 alone не demonstrable до A6 adapter. M2 (day 14) включает A6, что снимает риск.
- **Parser strictness** — regex для `- timestamp (high|med|low) statement` может быть хрупким при отклонениях LLM от формата. Mitigation: возвращаем `parse_error` reason вместо throw; A5/E мониторят `recall_usage_rate` / extraction frequency.
- **`PREV_OBS_WINDOW`** — default из §4.4 не зафиксирован числом; используем 4K tokens как разумный старт, calibratable в E.
- **Async buffer** в A5 заберёт `maybeExtractObservations` в pre-emptive режим; sync version остаётся для fallback и для testов.
