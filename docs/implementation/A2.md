## Core Transformation

Главный механизм tool-heavy policy: больше-чем-`T_SIZE` или cum-`T_CUM` tool_result атомарные группы offload'ятся в in-memory scratchpad с заменой на pointer placeholder + digest. Recall Tool инжектируется когда scratchpad непустой и `RECALL_TOOL=true` — definition стабильна (frozen literal, cache-prefix safe). §9.1 cache-invariance test промоутится с JSON.stringify proxy (A1) до bytewise через `serializeForCache(prefix): Uint8Array` с каноничным property-order.

Это даёт M1 cut-point (system_design §7.3): "AHC-type-only" работает на LongMemEval-Medium к концу day 7.

## Deviations

- **digest.ts `safeStringify` без `?? '""'`** — TS types гарантируют `string` return, ESLint strict triggered. Empty/undefined outputs ловятся upstream через `isEmptyOutput`, поэтому fallback не нужен. Документировано комментарием в коде.
- **`compactWithOffload` deps без `flags`** — `flags` берётся из `ctx.flags` (DigestDeps требует flags, но в orchestrator deps этот dup убран). Делает API caller'а лаконичнее.
- **`recallToolDefinition` без explicit `as ToolDefinition` cast** — `Object.freeze` возвращает `Readonly<T>`, который TS принимает как ToolDefinition через `Record<string, unknown>` структуру. Удалено per ESLint strict-no-unnecessary-type-assertion.
- **`CompactWithOffloadDeps` exposed как public** — план не выносил тип в public surface; добавлено для удобства caller'ов в A5.

## Risks / Follow-ups

- **`serializeForCache` property-order** — JS object iteration не гарантирует stable order при динамической сборке; explicit field-list serialization required, не `JSON.stringify`.
- **`T_SIZE_MIXED` design doc sync** — §2.4 не упоминает поле; sync `A_ahc-algorithm.md` отдельным docs PR.
- **Recall tool *execution* в A6** — core ships только definition; handler (lookup в scratchpad + возврат original tool_result) живёт в адаптере.
- **A3 потребует `LLMCaller` shape** — финализируем в Step 1; если в A3 понадобятся доп. поля (например, structured-output schema), расширяем минимально.
